Section 02

Design Philosophy

Editorial Minimalism

"Less noise, more trust. Every element must earn its place. We design with restraint, precision, and purpose — letting whitespace breathe and content speak."

Six Design Principles

Generous Whitespace
Space is not emptiness — it's clarity. Always use generous padding (80px–120px sections), let content breathe, and never crowd elements together.
Choreographed Reveals
Elements appear with staggered fadeUp animations and intentional delays. Each piece enters the stage purposefully — never all at once.
Muted Hierarchy
Use reduced opacity, light letter-spacing, and uppercase labels for supporting text. Headlines command attention; everything else defers gracefully.
Directional Cues
Subtle lines, gradients, and scroll indicators guide the eye. Use thin vertical or horizontal lines as visual anchors — never heavy borders or dividers.
Poppins Scaling
Poppins is our single type system — weight, size, spacing, and case create all hierarchy. Light (300) for body, SemiBold (600) for headings, Bold (700) for impact.
Restrained Color
Lead with neutrals and whitespace. Brand colors appear as deliberate accents — a teal CTA, a green divider, a subtle gradient — never as overwhelming fills.

Body Text Minimum — 15px Rule

All body text, card descriptions, list items, FAQ answers, and any content in <p> tags must be 15px minimum, with 16px as the standard. This applies to: card descriptions, step descriptions, serve/cost list items, FAQ answer text, callout descriptions, and all inline paragraph styles. Only labels (10–11px), badges, table cells (13px), subspecialty tags (12px), trust items (11px), and navigation elements are exempt from this minimum.

Logo Usage Rules

Always maintain minimum clear space around the logo. Use on clean, uncluttered backgrounds with sufficient contrast. Use approved white version on dark backgrounds. Never stretch, skew, distort, rotate, tilt, or apply effects like shadows or outlines. No custom logo recreations — use the official asset only.

Brand Color Mandate for All Imagery

Every image must strictly adhere to brand colors.

All generated, photographed, or designed imagery — including backgrounds, clothing, UI elements, charts, data visualizations, objects, and environments — must use only approved brand colors: Navy #0E143C, Teal #16C1C2, Aqua #4ECED1, Charcoal #222, Light Grey #F6F7FB, Grey #C6CCD8, and White #FFF. No off-brand colors are permitted anywhere within any image asset.

Image Placeholder Standards

Every page build must include image placeholder boxes exactly where images are needed — never skip or omit any image slot. Each page also ships with a separate Image Requirements file (.md) that specifies every image needed, its dimensions, purpose, brand color compliance rules, and editorial direction.

Every page build outputs exactly 2 files: (1) The production HTML page with all image placeholders present, and (2) a unified Image Requirements .md file. No exceptions — both files are mandatory deliverables.

Section 09

Content Integrity Mandate

This is the most critical rule in the entire brandbook. When redesigning or building any page, the content provided — whether from a document or pasted in chat — must be treated as sacred. The design is visual and structural only. The words are untouchable.

Absolute Content Protection Rule

Copy ALL source content verbatim — no additions, removals, rewording, or alterations of any kind. Every section in the source document must appear in the final page. No section may be missing, invented, merged, or split. This applies to headings, body copy, lists, FAQs, tables, workflow steps, and all text. The redesign is visual and structural only.

Content Handling Rules

✓ Content Rules — Required
✓Fetch or copy content exactly from source doc or chat
✓Every section provided must appear as its own designed section
✓Preserve all headings, subheadings, and hierarchy verbatim
✓Keep all list items, bullet points, and numbered steps intact
✓Retain all FAQ questions and answers word-for-word
✓Maintain all table data, rows, and columns as provided
✗ Content Rules — Forbidden
✗Rewording, paraphrasing, or "improving" any source text
✗Adding content, sections, or copy not in the source
✗Removing or omitting any section, paragraph, or detail
✗Merging separate sections into one combined section
✗Splitting one source section into multiple new sections
✗Changing terminology, brand names, or technical terms
Design transforms the visual experience. Content stays exactly as written.
If a source document has 12 sections, the final page must have 12 designed sections — no more, no fewer. The creative freedom is in layout, typography, spacing, animation, and visual treatment. The text is locked.

Section 10

Human Psychology Design Protocol

Every page must be designed through the lens of human psychology. Section order, visual weight, spacing, color application, and interaction patterns are driven by how healthcare decision-makers actually process, evaluate, and act on information — not by arbitrary aesthetics.

Cognitive Principles Applied Per Section

Authority & Trust First
Hero sections lead with credibility signals — compliance badges, acceptance rates, and clinical authority. Decision-makers evaluate trust before reading content. Establish it within the first viewport.
Progressive Disclosure
Information flows from broad (what we do) to specific (how we do it) to proof (results). This mirrors natural cognitive processing — reducing overwhelm while building conviction layer by layer.
Loss Aversion Framing
Content sections addressing pain points (denials, revenue leakage, compliance risk) are positioned before solution sections. Humans act more urgently to avoid loss than to acquire gain — design leverages this.
Social Proof Anchoring
Metrics, trust bars, and client outcomes are placed at psychological anchor points — immediately after the hero, between pain/solution sections, and before CTAs — where they reinforce credibility at decision moments.
Cognitive Load Reduction
Whitespace, muted hierarchy, and editorial minimalism reduce cognitive load. Healthcare decision-makers process complex information daily — our layouts give their eyes breathing room and clear visual paths.
Action-Ready CTA Placement
CTA sections appear after the reader has processed enough information to feel confident acting. Never too early (before trust), never too late (after attention fades). Position at the psychological "ready" moment.

Section Design Psychology Workflow

StepPsychology Action
1. Read ContentAnalyze every section of provided content — understand what each communicates and what emotional/logical response it should trigger in the reader.
2. Map User JourneySequence sections to follow the reader's psychological decision path: awareness → pain recognition → solution understanding → proof/trust → action readiness.
3. Assign Visual WeightHigher-impact sections (hero, core services, CTA) receive more visual prominence. Supporting sections (compliance, subspecialties) use lighter visual treatment.
4. Apply Contrast RhythmAlternate between light/dark backgrounds, dense/sparse layouts, and text-heavy/visual sections to maintain cognitive engagement and prevent fatigue.
5. Validate EmotionallyReview the complete page as if seeing it for the first time. Does each section feel purposeful? Does the page build confidence? Is the CTA positioned at the moment of maximum conviction?
Design is not decoration — it is persuasion architecture.
Every section layout, spacing choice, color application, and animation timing must serve a psychological purpose. If a design decision cannot be justified by how it affects the reader's decision-making process, it does not belong on the page.
